# Project Contribution Summary

**Objective:** Generate a concise, professional summary of a user's contributions to a software project based on their technical involvement and impact metrics.

**Instructions:**

You are analyzing a user's contributions to a project. Based on the provided data, create a compelling 2-3 sentence summary that highlights:

1. The primary technologies and tools used (programming languages, frameworks)
2. The user's level of contribution (commit percentage, lines changed)
3. The project type and any notable aspects

Guidelines:
- Be concise and professional (suitable for a portfolio or resume)
- Focus on technical achievements and quantifiable contributions
- Use active voice and highlight impact
- Mention key technologies prominently
- If contribution is high (>50%), emphasize leadership or primary development role
- If contribution is moderate (20-50%), emphasize collaborative development
- If contribution is low (<20%), emphasize specific contributions or learning

Project Data Format:
- Project Name: [name]
- Classification: [coding/writing/art/mixed]
- Primary Languages: [list]
- Frameworks: [list]
- Contribution Score: [0-100]
- Commit Percentage: [percentage]
- Lines Changed Percentage: [percentage]
- Total Commits: [number]
- Date Range: [timestamp]

**Expected Output:** A 2-3 sentence professional summary highlighting the user's technical contributions, technologies used, and impact level.
